Warum unterscheiden wir zwischen unterschiedlichen Datentypen?
  • Effizienz: Unterschiedliche Datentypen ermöglichen die effiziente Speicherung und Verarbeitung von Informationen je nach Bedarf, wodurch Ressourcen optimal genutzt werden.
  • Klarheit und Lesbarkeit: Durch die Verwendung von verschiedenen Datentypen können Programmierer den Code leichter lesen und verstehen, da sie Informationen über die Art der gespeicherten Daten erhalten.
  • Präzision: Verschiedene Datentypen ermöglichen die präzise Darstellung von Informationen. Zum Beispiel können Fließkommazahlen Dezimalstellen enthalten, während Ganzzahlen dies nicht tun.
  • Validierung: Datentypen können zur Validierung von Benutzereingaben verwendet werden, um sicherzustellen, dass die erwarteten Datenformate und -werte eingehalten werden.
  • Berechnungen: Unterschiedliche Datentypen sind für unterschiedliche Arten von Berechnungen und Operationen geeignet. Ganzzahlen werden beispielsweise für einfache arithmetische Operationen verwendet, während Booleans für logische Operationen verwendet werden.
  • Programmfluss: Datentypen beeinflussen den Programmfluss, da sie Bedingungen und Schleifen steuern. Zum Beispiel kann ein Boolean-Wert dazu verwendet werden, eine bedingte Anweisung zu aktivieren oder zu deaktivieren.
  • Datensicherheit: Die Verwendung geeigneter Datentypen kann dazu beitragen, Datenfehler und Sicherheitslücken zu verhindern, indem sie sicherstellen, dass Daten in einem erwarteten Format vorliegen.
  • Interoperabilität: Verschiedene Datentypen sind in verschiedenen Kontexten und Technologien nützlich und können zur Interaktion mit verschiedenen Systemen und APIs verwendet werden.
  • Fehlererkennung und Debugging: Die Verwendung von Datentypen hilft bei der frühzeitigen Erkennung von Fehlern im Code, da ungültige Datentypen oder Konvertierungsprobleme leichter erkennbar sind.
  • Dokumentation: Die Verwendung von Datentypen in Variablendeklarationen und Funktionssignaturen dient als Dokumentation, die anderen Entwicklern hilft, den Code zu verstehen und korrekt zu verwenden.

Modules / Libraries

Was ist ein Python Module / Library?
  • Eine Sammlung von "Scripts", die einem helfen spezifische Aufgaben einfacher zu erledigen
  • Beispiel Pandas. Base-python hat keinen umgang mit tabellarsichen Daten. Um dies zu erleichtern, wurde Pandas geschieben.
  • Mit Pandas kann man tabelarrische Daten (z.B. CSVs) einlesen, darstellen, manipulieren und erweitern. Entspricht einem MS Excel in Python
  • Häufig sind diese Scripts ebenfalls in Python geschrieben, aber nicht immer
Woher werden diese "Scripts" geholt und wo werden sie abgespeichert, um sie nutzbar zu machen?
Also wo werden die scripts installiert?
  • es ist wichtig, dass man dieses konzept versteht!
  • Zuerst liegt das Script irgendwo auf einem server, zb bei conda-forge
  • Um es nutzen zu können, muss man es zuerst installieren. So wird das skript irgendwo auf dem computer gespeichert
  • Um es zu nutzen, muss man es in die jeweilige session laden
  • Analog Bibliothek: Buch ausleihen, Buch zur Hand nehmen
  • Wo werden die scripts installiert? Hier kommen Virtual Environments ins spiel

Schritt 3: Eine Virtual Environment erstellen

(Falls du letzte Woche bereits einen Environment namens geopython erstellt hast, wähle einen anderen Namen)

conda create --name geopython

  • Ruft das Programm conda auf
  • Ruft den Befehl create von conda auf
  • Kündigt den Namen der neuen Umgebung an
  • Bestimmt der Name der neuen Umgebung (beliebig)
  • "Create" entspricht einer "Funktion" aus dem Programm conda
  • --name ist ein Argument dieser Funktion
  • geopython ist der Wert dieses Arguments
Speicherort: C:\Users\rata\AppData\Local\ESRI\conda\envs mit Y+Enter bestätigen
Wenn wir den entsprechenden Pfad im Windows Explorer anschauen sehen wir, dass conda einen neuen Ordner erstellt hat, der den Namen der Environment trägt
Der Folder ist aber noch ziemlich leer, weil wir noch keine Module installiert haben.

Schritt 4: Environment aktivieren

# Conda version > 4.6
conda activate geopython

# Conda version <= 4.6
activate geopython

Aktiviert die Environment geopython

Schritt 5: Modul installieren

conda install -c conda-forge pandas
  • Ruft das Programm condaauf
  • Ruft den Befehl installvon conda auf
  • Kündigt der Name des channels an (von wo soll das Modul installiert werden?)
  • Ist der Name des channels
  • Ist der Name des Python Moduls
  • Conda fragt uns, ob wir das Modul inkl. dependencies installieren wollen
  • Wir bestätigen mit Y+Enter
Es dauert eine Weile, bis das Modul inkl. den dependencies installiert sind.

Für Programmieren II brauchen wir nachstehende Module von conda-forge:

  • jupyterlab
  • pandas
Sobald jupyterlab installiert ist, könnt ihr die Anwendung mit folgendem Befehl in die Konsole starten: jupyter lab
